Overview

Welcome to Mock Market, a playful yet data-driven stock analysis demo that uses a blunt, entertaining style. The idea is to let you pick a few S&P 500 stocks, generate real historical data, and see an AI-generated opinion on whether it’s time to buy, sell, or hold.

Below, you’ll find more details on the background animation, our approach with various libraries, the Polygon and OpenAI calls, and our responsibility disclaimer.

Background Animation

The floating-line animation you see behind everything is adapted from a CodePen snippet by @tati420. We adjusted the spacing, color scheme, and velocity to fit the vibe of the app while maintaining that subtle, techy background feel. No performance issues were observed even on mobile devices, making it a fun but lightweight visual.

Libraries Used

  • Chart.js & date-fns Adapter: For rendering simple line charts of each stock’s performance. The date-fns adapter ensures we can parse timestamps quickly and display them as monthly intervals.
  • OpenAI Node/Chat API: For obtaining witty, no-BS stock recommendations. We feed it the cleaned historical data and retrieve structured text or JSON back.
  • Polygon.io API: For fetching the daily historical prices, volumes, highs, and lows of each chosen stock.
  • Local JSON: A local list of S&P 500 ticker symbols is used for quick ticker lookups, so we don’t hammer external autocomplete APIs.

Polygon API

We call the Polygon.io endpoints (via serverless functions on Vercel) to gather real historical data for the chosen tickers: daily closing price, volume, high, low, etc. This approach keeps your front end clean and ensures sensitive API tokens remain on the back end. Once that data returns, we format it to highlight each stock’s relevant stats (yearly high, low, recent average volume, etc.).

OpenAI Usage

After Polygon returns the numbers, we send them off to OpenAI through a custom prompt that requests an honest, slightly irreverent commentary. The conversation is structured to produce buy/hold/sell recommendations without disclaimers or random disclaimers (unless we want them). We also instruct OpenAI to return a short overview, plus a bullet for each ticker.

Additionally, we gather an integer score (0–100) from the AI’s response (depending on prompt), which we map to our “sell/hold/buy” bar. That’s purely for user visualization—there’s no official or universal scoring system behind it.

Data Processing

Before sending the historical data to OpenAI, we streamline it into a set of bullet points indicating the stock’s 1-year performance change, highest/lowest closes, average volume, and current price. That way, the prompt remains concise, and the AI can quickly grasp each stock’s key info.

We also store the user’s chosen tickers so we can place them back into the final text output. The local JSON file with S&P 500 data ensures the front-end searching is fast and offline-friendly.

Graphs & Charts

Each ticker’s performance is visualized below its AI analysis, courtesy of Chart.js. We fetch all daily close prices, then let Chart.js turn them into a simple line with minimal fuss—just enough to see the broad up/down trend. We also use an adapter (e.g. date-fns) for time parsing, ensuring the X-axis shows readable month/year labels.
